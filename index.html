02 / 04 Core Principles

Four pillars of Bebop motion

A unified motion language adapted for Bebop's visual identity and component library.

01

Informative

Motion builds a spatial mental model. Transitions reveal where elements come from and where they go. The drawer slides from the left because that's where it lives. Snackbars enter from their anchored edge.

Drawer slide-in Snackbar positioning Nav transitions

Bebop Example

BebopDrawerScaffold — Drawer slides from the left edge via animateFloatAsState with tween(250). A scrim fades proportionally to the drawer's progress, reinforcing the layered spatial model.

02

Focused

Animation draws the eye to what matters without creating distraction. Micro-interactions confirm user actions instantly. Larger transitions guide the eye smoothly during navigation context changes.

Toggle thumb Button morph Search icon fade

Bebop Example

BebopToggle — Thumb slides with animateDpAsState(tween(200)). Fast enough to feel responsive (200ms), slow enough for the eye to track the state change.

03

Expressive

Motion conveys personality. Spring physics add life to enter animations, while controlled tween exits keep departures crisp. The system feels alive but never chaotic or overanimated.

Spring bouncy enter Content crossfade Scrim alpha ramp

Bebop Example

BebopSearchBar — Filter chips appear with spring(DampingRatioLowBouncy, StiffnessLow) + fadeIn(tween(350)). A playful overshoot that settles naturally.

04

Consistent

Shared timing tokens ensure similar interactions always feel identical. A fade-out is 150-250ms. An enter is 250-350ms. Same curves, same durations, same feel — across every component.

Duration tokens Easing curves Pattern reuse

Bebop Example

System-wide — Every slide+fade combo across Drawer, Snackbar, and SearchBar shares the same tween durations. Changing a token value updates the entire system at once.

03 / 04 Easing & Duration

Timing & easing tokens

Standardised curves and durations that every Bebop component references. Tuned for our components and interaction patterns.

Standard

EaseInOut

Default curve. Smooth start and end for natural feel.

cubic-bezier(0.4, 0, 0.2, 1)

Decelerate

Ease Out

Enter animations. Arrives fast, then settles.

cubic-bezier(0, 0, 0.2, 1)

Accelerate

Ease In

Exit animations. Starts slow, accelerates away.

cubic-bezier(0.4, 0, 1, 1)

Playful

Spring (Bouncy)

Expressive enters. Overshoots, then settles.

Spring(LowBouncy, LowStiffness)

Rule of asymmetry: Exits are always faster than enters (150-250ms vs 250-350ms). Users want instant feedback when dismissing, but need a moment to register new content appearing on screen.

04 / 04 Transition Patterns

Patterns in practice

Five transition patterns used across Bebop components, with live visuals and implementation examples.

Pattern 1

Slide + Fade

Slides in from a directional origin while fading. Gives spatial context — the user knows where content lives.

Drawer Snackbar Search chips
// Snackbar enter/exit
enter = slideInVertically(tween(300)) { -it }
      + fadeIn(tween(300))
exit  = slideOutVertically(tween(250)) { -it }
      + fadeOut(tween(250))

Pattern 2

Fade Through

Outgoing fades out, incoming fades in. For elements sharing the same space across different states.

Button preview swap Parameter changes
// Button showcase content swap
AnimatedContent(targetState) {
  transitionSpec = {
    fadeIn(tween(300)) togetherWith
      fadeOut(tween(300))
  }
}

Pattern 3

Spring Enter

Physics-based spring that overshoots and settles. Adds personality and delight to element entrances.

Search filter chips Expandable sections
// Search bar bouncy chip entrance
enter = expandHorizontally(
  spring(
    dampingRatio = Spring.DampingRatioLowBouncy,
    stiffness = Spring.StiffnessLow
  )
) + fadeIn(tween(350))

Pattern 4

Value Animation

Continuous animation of individual properties — position, opacity, color, size. The workhorse of micro-interactions.

Toggle thumb Drawer progress Scrim alpha
// Toggle thumb position
val thumbOffset by animateDpAsState(
  targetValue = if (checked) 20.dp else 0.dp,
  animationSpec = tween(200)
)

Pattern 5

Scrim + Overlay

Semi-transparent overlay fades behind modal content. Alpha is driven by an animated float, seamlessly linking scrim dimming to content animation progress.

Scrim animates with content
Drawer scrim Dialog backdrop
// Drawer scrim — alpha tied to progress
val progress by animateFloatAsState(
  targetValue = if (isOpen) 1f else 0f,
  animationSpec = tween(250)
)

// Scrim applies animated alpha
Box(Modifier.background(
  Color.Black.copy(alpha = 0.4f * progress)
))

Choosing the right pattern

Scenario Pattern Why
Element enters from an edge Slide + Fade Spatial context — user sees the origin
Content swaps in same container Fade Through Smooth transition without spatial shift
Delightful, personality-driven enter Spring Enter Adds life without being distracting
Component property changes state Value Animation Continuous, interruptible, composable
Modal or overlay appears Scrim + Overlay Dims background, focuses attention on content

Implementation note: All Bebop animations use declarative animation APIs — AnimatedVisibility, animate*AsState, AnimatedContent. We avoid imperative animation controllers. Let the framework handle interruption and state management.
